## プライオリティマトリクス フォーマット規則（必須）

Phase 3（ユースケースマッピング）の `## 3-x. プライオリティマトリクス` セクションは、**必ずMarkdown テーブル形式**で生成すること。ASCII art（code block内の罫線図）は使用禁止。

### フォーマット選択基準

| 形式 | 使用場面 | 例 |
|:---:|---|---|
| **Format A**（高/中/低×実現可能性） | 標準的な戦略価値×実現可能性マトリクス | 金融・製薬・電力・製造業（標準） |
| **Format B**（高/低インパクト×難易度） | 4象限インパクト×難易度マトリクス | 保険・化学品 |
| **Format C**（2×2 ビジネスインパクト×実現性） | Quick Win/Strategic Bet/Low Hanging Fruit/Deprioritize | 汎用製造業 |

### Format A テンプレート

```markdown
| 戦略価値 | 実現可能性：低（実現困難） | 実現可能性：高（実現容易） |
|:---:|:---|:---|
| **高** | UC番号 名称 | **[Quick Win] UC番号 名称**<br>UC番号 名称 |
| **中** | UC番号 名称 | UC番号 名称<br>UC番号 名称 |
| **低** | UC番号 名称 | UC番号 名称 |
```

### Format B テンプレート

```markdown
| インパクト | 難易度：高（実現困難） | 難易度：低（実現容易） |
|:---:|:---|:---|
| **高インパクト**<br>戦略的優先度：高 | UC番号 名称 | UC番号 名称 |
| **低インパクト**<br>Quick Win領域 | UC番号 名称 | **[Quick Win] UC番号 名称** |
```

### Format C テンプレート

```markdown
| 実現性 | ビジネスインパクト：低 | ビジネスインパクト：高 |
|:---:|:---|:---|
| **実現容易**<br>Low Hanging Fruit / Quick Win | ・名称 | **[Quick Win] ・名称** |
| **実現困難**<br>Deprioritize / Strategic Bet | ・名称 | *・名称* |
```

### Quick Win 判定ルール
- **位置ベース**: 右列（実現容易）または bot_right 象限のユースケースのみ `**[Quick Win] ...**` でボールド
- 左列（実現困難）はたとえ★★★★★評価でもQuick Winとしない
- UC見出しに `★★★★★` かつ `[最優先]` または `[Quick Win]` がある場合に強調表示
- 複合ラベル `[最優先Quick Win]` も検出対象
