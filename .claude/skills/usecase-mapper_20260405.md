# バリューチェーンAI機会マッピング

対象: `$ARGUMENTS`

## 実行内容

### 3-1. 機会の特定（バリューチェーンの各工程ごと）

対象企業/業界のバリューチェーンを特定し、各工程について以下を評価:

| 評価項目 | 内容 |
|---------|------|
| 現在の課題/ペインポイント | 具体的な問題を記述 |
| 適用可能な生成AIユースケース | 具体的なAI活用方法 |
| 想定効果（定量） | 金額、%、時間削減の数値 |
| 技術的実現可能性 | 高/中/低 |
| 導入難易度 | 高/中/低 |

### 3-2. プライオリティマトリクス（2x2）

```
              ビジネスインパクト
              低          高
実  容易  | Low Hanging  | Quick Win    |
現         | Fruit       | （即座着手） |
容  困難  | Deprioritize | Strategic    |
易         | （見送り）   | Bet（投資判断）|
性
```

各ユースケースを4象限に配置し、優先順位を決定。

### 3-3. トップ10ユースケース詳細

優先度上位10件について以下を記載:
- ユースケース名と概要（1行）
- 対象部門/プロセス
- 期待効果（KPI、金額、%）
- 必要な技術スタック
- 想定投資額と回収期間
- リスクと緩和策
- 他社の成功事例（企業名、成果の数値）

## 出力形式
- `output/{対象名}/phase3_usecase_mapping.md` に保存
- すべての数値にフェルミ推定の根拠を明記
