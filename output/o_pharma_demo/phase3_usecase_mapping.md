> 作成日: 2026年4月14日

---

> **このセクションの要点**
> - 全26ユースケースのうち、Quick Win最優先はUC21（MR支援AI）・UC22（薬事CTD生成AI）・UC23（ナレッジ管理AI）の3件。Year 1から着手し、Month 12までにROI可視化
> - 最高戦略価値はUC1（CNS創薬AI）とUC3（OT oncology AI）。投資回収は3-5年だが、パイプライン価値換算で100億円超のオプション価値がある
> - GxP対象ユースケース（UC4-UC8）はPMDA事前相談と計算機化システムバリデーション（CSV）が前提。Phase 0から規制対応を並行して進めることが全体スケジュールの鍵

## 3-2. Top 10 ユースケース詳細

### Quick Win Group（Year 1-2 着手）

#### UC21：MR支援AIアシスタント ★★★★★ [最優先]

| 項目 | 内容 |
|-----|-----|
| **概要** | 国内MR約3,000名向けの医療情報提供・提案書生成AIアシスタント |
| **主機能** | 学術論文サマリー生成、処方医ニーズ予測、訪問記録自動作成、製品Q&A |
| **ベース技術** | Claude Sonnet 4.6（日本語） + 社内薬事承認情報RAG |
| **Year 1効果** | 資料作成工数▲55%、MR一人当たり訪問数+20%、成約率+8% |
| **3年累計ROI** | 約15億円（MR約3,000名×時間価値換算） |
| **規制分類** | Layer A（非GxP）。ただし医薬品情報提供ガイドラインへの準拠必須 |
| **着手時期** | Month 5（パイロット30名）→ Month 9（全展開） |

#### UC22：薬事申請AI（CTD自動生成・照会対応） ★★★★★ [最優先]

| 項目 | 内容 |
|-----|-----|
| **概要** | 新薬承認申請書（CTD）の構成・文書草案生成AIと、PMDA照会事項への自動回答支援 |
| **主機能** | 治験総括報告書（CSR）要約、CTDモジュール構成支援、照会事項分類・回答テンプレート生成 |
| **ベース技術** | Claude Opus 4.6（複雑薬事文書の品質が必要） + 薬事RAGデータベース |
| **Year 1効果** | CTD作成工数▲45%（年間2-3申請×平均800時間工数削減） |
| **3年累計ROI** | 10億円（工数削減+承認期間短縮による市場投入加速） |
| **規制分類** | Layer B（GxP隣接）。AI出力は薬事担当者が必ず確認・修正 |
| **着手時期** | Month 5（パイロット：進行中1申請で試験適用） |

#### UC23：社内ナレッジ管理AI（製薬特化RAG） ★★★★☆ [最優先]

| 項目 | 内容 |
|-----|-----|
| **概要** | 社内SOP・治験計画書・モニタリング記録・学術文献の統合ナレッジAI |
| **主機能** | SOP検索・要約、社内事例の横断検索、新人研修支援 |
| **ベース技術** | Azure OpenAI（RAG） + Azure Cognitive Search |
| **Year 1効果** | 情報探索工数▲40%（3,500名内勤） |
| **3年累計ROI** | 6億円 |
| **着手時期** | Month 4（Phase 0内で先行構築） |
