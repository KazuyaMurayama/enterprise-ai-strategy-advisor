## 1. プロジェクトの目的と本日の位置づけ

**本プロジェクトは「人員削減計画」ではなく「業務再定義プロジェクト」です。**

| 一般的アプローチ（本プロジェクトでは採用しない） | 本プロジェクトのアプローチ |
|---|---|
| 削減目標（▲7名）を先に決める | 業務の性質を先に定義する |
| 削減できる業務を後から探す | 人間が必須な業務を先に守る |
| 現場は削減される側に立つ | 現場が業務定義の主体者になる |
| 結論先行で役員会承認を求める | 根拠先行で役員会承認を求める |

**本日の役員会でお願いすること**: 「人員削減の承認」**ではなく**「判断業務の定義の承認」です。

## 2. 判断業務15項目の定義（★本日の承認対象）

### 3軸スコアリング評価モデル（概要）

```
軸A: 情報完全性  ─ 判断に必要な情報がデータとして揃っているか（0〜3点）
軸B: 判断再現性  ─ 同じ情報で誰でも同じ判断ができるか（0〜3点）
軸C: 例外頻度    ─ 想定外の状況がどのくらい発生するか（0〜3点）
                                                       合計9点満点

  0〜2点 → 実行業務（AI/RPA代替可能）
  3〜4点 → 準判断業務（AI提案＋人間承認）
  5〜9点 → 判断業務（人間必須）   ← 本日承認対象
```
