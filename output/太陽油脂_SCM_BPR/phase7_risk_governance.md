## 7-2. 最重要リスク対処計画

### R-01: 労務リスク（最優先）

**シナリオ**: 削減対象者または組合が「恣意的な削減」として異議申し立てを行う

```
予防策（Phase 3が根本的解決策）:
  ① 3軸スコアリング結果を全担当者に開示・説明
  ② 各担当者が自分の業務スコアに異議を言える場（ヒアリング）を設ける
  ③ 役員会での正式承認により「客観的分類プロセスの存在」を記録に残す
  ④ 削減は「業務のなくなり方」であり「人の価値の否定」ではないことを明示

対応プロセス（問題発生時）:
  発生 → 人事部・法務部に即時報告
      → スコアリング根拠文書を証拠として提示
      → 業務再定義プロセスの透明性を示す
      → 外部アドバイザーの独立した分析として第三者性を強調

記録保管:
  Phase 3スコアリング原票・ヒアリング記録を5年間保管
  役員会議事録に判断業務定義の承認を明記
```

### R-02: 親会社物流ユニット調整不調

**シナリオ**: 親会社物流ユニットが業務移管・棲み分けに合意せず、物流チーム▲3名が実行不能

```
発生条件: 6月末までに合意が取れない場合
影響: 物流チーム削減を▲3名→▲1名に縮小（合計▲7名→▲5名）

軽減策:
  ① 交渉開始は「5月20日役員会承認後直ちに」をコミット（承認項目②）
  ② 役員レベルでの支援要請（CFO/COO経由での親会社上位意思決定者への打診）
  ③ 交渉が長期化する場合でも「B-01,02,03の自動化のみ」で▲2名を確保できる設計を保持

フォールバック計画:
  親会社合意が取れない場合 → 外注化1,200h分をB-01〜B-03の自動化深度向上で補完
  → 削減数は▲3名→▲2名に変更し、全体は▲7名→▲6名でも十分なROI
```

## 7-3. 現場の心理的障壁と変革管理

### 担当者が抱く恐怖のパターン

| 恐怖のタイプ | 典型的な言葉 | 実態 | 対処方針 |
|---|---|---|---|
| **役割消失恐怖** | 「自分の仕事がAIに取られる」 | 実行業務は移行、判断業務は保護 | スコアリング結果の個別説明 |
| **評価低下恐怖** | 「自動化されたら自分の価値が下がる」 | AI管理能力が新たな評価軸に | 新評価基準の事前開示 |
| **学習不安** | 「新しいシステムを使いこなせるか」 | 研修プログラムで対応可能 | 段階的研修・サポート体制明示 |
| **将来不安** | 「次の削減も自分では」 | TO-BE 19名体制を公式コミット | 役員会承認後の公式発表 |

### Reframingメッセージ（全担当者向け）

```
【言ってはいけないメッセージ】
  「AIで業務効率化するため、一部の人員は不要になります」

【正しいメッセージ】
  「皆さんが今こなしている『単純な入力・集計・転記』という
   時間のかかる作業をAIに移行することで、
   皆さんは『この会社の事業判断に本当に必要な仕事』に
   集中できる体制を作ります。

   今日、この役員会で確認したいのは：
   『どの業務が皆さんにしかできない判断業務なのか』
   という定義です。」
```

### チェンジマネジメントの4段階

| 段階 | 時期 | アクション | 成功指標 |
|:---:|:---:|---|---|
| **認識** | 5月〜6月 | 全体説明会（Reframingメッセージ）・個別ヒアリング | 担当者の「なぜ」への回答率100% |
| **受容** | 6月〜7月 | スコアリング結果の個別開示・異議プロセスの透明化 | スコア異議件数の収束 |
| **準備** | 7月〜9月 | 新システム研修・役割再設計のワークショップ | 研修参加率95%以上 |
| **定着** | 10月〜 | 月次レビュー・成功事例の共有・評価制度適用 | 90日後の定着率確認 |

## 7-4. AI/RPAガバナンス機構

### 月次精度監査（運用開始後）

| 監査項目 | 担当 | 基準値 | 対処 |
|---|:---:|:---:|---|
| OCR受注入力の自動処理率 | S-01担当者 | 75%以上 | 閾値調整・再学習 |
| OCR受注入力の誤認識率 | S-01担当者 | 1%以下 | モデル再訓練 |
| 配車AIの最適化達成率 | S-01担当者 | 85%以上 | パラメータ調整 |
| MRP自動化の例外発生率 | IT部門 | 5%以下 | ルール見直し |
| 生成AIの判断介入率 | 各チーム長 | 5%以下 | プロンプト改善 |

### 四半期再評価プロセス

```
業務分類は固定ではない。以下の事由がある場合は再評価を実施:

  ① 新技術の登場: スコアの下方修正（より自動化しやすくなった）
  ② 事業環境変化: スコアの上方修正（例外頻度が増した）
  ③ 精度実績の更新: 自動化率が目標超過した場合（追加削減余地の確認）

再評価実施タイミング: 毎年1月・4月・7月・10月
再評価担当: S-01（AI運用管理担当）+ 各チーム長
承認: 業務改革部長
```

### AI倫理・コンプライアンスチェックリスト

- [ ] 生成AI APIに個人情報（顧客名・住所）を含むデータを送信していないか
- [ ] AI判断の結果を人間がレビューできるログが保存されているか（監査可能性）
- [ ] 自動処理の誤りによる顧客への損害発生時の補償プロセスが定義されているか
- [ ] AI判断による差別・不公平な扱いが発生していないか（特に受注優先度付け）
- [ ] システム障害時に手動バックアップへ切り替えられる手順が整備されているか
