## 6-3. ステークホルダー管理

### ステークホルダーマップ

```
               高
               ↑
      関与度    │    【A: 重点管理】          【B: 緊密連携】
               │    ・役員会（CFO/COO）       ・業務改革部長
               │    ・親会社物流ユニット長    ・IT部門長
               │
               │    【C: 情報共有】           【D: モニタリング】
               │    ・現場担当者（26名）       ・労組（存在する場合）
               │    ・SIer担当者              ・外部監査
               └──────────────────────────────→
               低        影響力        高
```

### ステークホルダー別対応戦略

| SH | 役割 | 懸念事項 | 対応方針 | 接触頻度 |
|---|:---:|---|---|:---:|
| **CFO** | 最終承認者 | 投資回収の確実性 | ROI計算根拠・感度分析を提示 | 月1回 |
| **COO** | 事業継続責任者 | 業務品質の維持 | 判断業務保護15項目を先に説明 | 月1回 |
| **業務改革部長** | プロジェクトオーナー | 現場の反発 | 週次進捗共有・現場代弁役として機能 | 週1回 |
| **各チーム長** | 現場責任者 | 人員削減への不安 | スコアリング結果を共同レビュー | 隔週 |
| **親会社物流ユニット長** | 外部決定権者 | 業務移管への抵抗 | B-07の影響範囲を限定的に提示 | 月2回 |
| **IT部門長** | 技術実装責任者 | システム複雑性 | 段階的実装・リスク分散を提示 | 週1回 |
| **現場担当者26名** | 直接影響者 | 自分の仕事がなくなる | Reframing：役割変化・スキルアップ機会 | 随時 |

### 重点ステークホルダー: 親会社物流ユニット

```
なぜ最重要外部SHか:
  → 物流チーム▲3名のうち1名分は「外注化」と「親会社との業務棲み分け見直し」が前提
  → B-07（親会社物流ユニットとの業務調整）が合意なしでは物流削減が実行不能

交渉戦略:
  Step1（5月〜6月）: 現状の業務重複・非効率を数値化して問題共有
  Step2（6月）: 「削減」ではなく「業務の最適分担」として提案
  Step3（6月末）: 覚書（MOU）締結による合意確定

交渉上のReframing:
  【言ってはいけない】 「御社との業務を削減します」
  【言うべきこと】     「御社の強み（物流ネットワーク）を活かせる業務に集中していただき、
                        当社はSCMの頭脳部分（需給調整・計画）に注力します」
```
