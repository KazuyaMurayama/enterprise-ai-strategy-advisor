## 4-1. 技術適用マトリクス概要

### 自動化技術の選択肢

| 技術カテゴリ | 代表ツール | 適用条件 | 実装難度 | コスト感 |
|---|---|---|:---:|---|
| **RPA（定型業務自動化）** | UiPath / Power Automate | 画面操作・定型フォーム処理 | 低 | 300〜500万/ライセンス |
| **OCR+RPA（文書入力）** | ABBYY FlexiCapture + RPA | 紙・FAX・メールPDFの文字認識 | 中 | 500〜800万（初期） |
| **生成AI（文書生成・分析）** | Claude API / GPT-4o API | 非構造化テキスト処理・要約・提案 | 中 | API従量課金 100〜300万/年 |
| **生成AI+最適化エンジン** | Claude + OR-Tools / Gurobi | 複合最適化（配車・生産計画） | 高 | 1,000〜2,000万（初期） |
| **BIツール（レポート自動化）** | Power BI / Tableau | 集計・可視化・定期配信 | 低 | 100〜200万/年 |

### 太陽油脂固有の制約条件（自動化難度に影響）

| 制約 | 内容 | 影響する業務 |
|---|---|---|
| バッチ製造の歩留まり変動 | ±5〜10%の変動が常態、確定値が遅延する | G-01,G-03,B-01 |
| 共同製品・副産物の価値不確実性 | 主製品収率により副産物量が変動 | G-05（高度判断のまま） |
| OEM仕様の多様性 | 顧客ごとに仕様書フォーマットが異なる | J-05 OCRの難度UP |
| FAX受注の割合（約40%） | 日本の取引慣行上FAX廃止困難 | J-01 OCR化の前提 |
| 親会社物流との共用システム | システム変更に親会社承認が必要 | B-01〜B-03 改修の制約 |

**実装技術詳細: G-01標準品MRP自動化**

```
現在の業務フロー:
  ERP照会（在庫・BOM） → Excel転記 → 計算式手入力 → 発注量決定 → 発注入力

自動化後フロー:
  ERP自動照会 → RPA計算実行 → 閾値チェック → 自動発注入力 → 担当者への完了通知
  ※例外（在庫異常・リードタイム変動）は人間にエスカレーション

実現可能性根拠:
  - 標準品BOMは安定（変更頻度: 月0〜1回）
  - 発注量計算ロジックは Excel VBAで既に定式化済み
  - ERP（SAP）はAPI連携可能
  推定工数削減率: 90%（残10%は例外対応）
```

> ※B-01〜B-03の3,000hは物流チーム▲3名の主要削減源。加えて**一部外注化**（定型配送業務 1,200h相当）で合計4,200h削減を実現。

**実装技術詳細: B-02配車最適化AI**

```
現在の課題:
  配送先150〜200箇所/日を担当者が経験と勘で配車
  → 平均配送コスト: 業界水準比 約12%高

AI化アプローチ:
  Google OR-Tools（オープンソース最適化ライブラリ）を活用した
  VRP（Vehicle Routing Problem）解法
  → 制約条件: 積載量・配送時間窓・車両台数・ドライバー勤務規制
  → 出力: 最適配車ルート案（5〜10分で計算完了）
  → 人間確認: 非定型顧客・初回配送・特殊条件時のみ

期待効果:
  - 配車工数: 960h/年 → ほぼゼロ（例外対応30h残存）
  - 配送コスト削減: 8〜10%（推定 年間 500〜700万円）
  ※配送コスト削減効果は人件費削減と別計上
```

**実装技術詳細: J-01 OCR+RPA（★最大削減インパクト）**

```
現在の受注フロー:
  FAX受信（40%）・メールPDF（35%）・EDI（25%） → 人手でシステム入力（年間2,400h）

OCR+RPA化フロー:
  ① FAX受信 → FAXサーバー → PDF自動保存
  ② ABBYY FlexiCapture でOCR処理（顧客コード・品番・数量・納期を抽出）
  ③ 信頼度スコア判定:
       95%以上 → 自動でERP入力・受注確認メール送信
       70〜94% → 担当者に確認依頼（5秒確認で完了）
       70%未満 → 従来通り手入力（例外処理）
  ④ RPA がERP画面を操作して受注登録

期待値:
  現状2,400h/年 → 自動処理率75%想定 → 担当者工数 600h/年（75%削減）
  信頼度95%超の自動処理率は他社FAX-OCR事例より（製造業平均70〜80%）
  太陽油脂では顧客・品番が限定的（約300社×約2,000品番）なため、学習後は精度向上期待
```
