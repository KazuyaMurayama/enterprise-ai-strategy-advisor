# フェーズ4: 技術アーキテクチャ戦略 — 四国電力株式会社

> 作成日: 2026年4月14日

---

> **このセクションの要点**
> - 四国電力のAIアーキテクチャは「OT（運用技術）系AI」と「IT系AI」の分離が最重要設計原則。発電所・変電所の制御システム（SCADA/DCS）とAIを直接接続することは安全基準上不可で、OT/ITブリッジを介した安全な設計が必須
> - 最大の技術課題は「発電所・変電所の老朽化センサーとIoT基盤の整備」。AIが学習するデータの品質が全予知保全AI・需給AIの精度を決定する。センサー更新投資はAI投資と一体で計画する
> - 基盤モデルはコールセンター・法人営業AIにClaude Sonnet 4.6、需給予測・設備AIは専門の時系列MLモデルを活用。エネルギーAI特化プラットフォーム（OSIsoft PI/Azure Energy等）との統合が鍵

---

## 4-1. 電力AI 3層アーキテクチャ

```
┌──────────────────────────────────────────────────────────────┐
│  Layer A: 業務・顧客AI（IT系）                                 │
│  [コールセンターAI] [法人営業AI] [ナレッジ管理AI]               │
│  [書類作成AI] [競合情報AI] [HR AI] [財務AI]                    │
│  ↑ Claude Sonnet 4.6 API / Azure OpenAI + RAG基盤            │
├──────────────────────────────────────────────────────────────┤
│  Layer B: エネルギービジネスAI（IT/OTブリッジ）                  │
│  [電力需給予測AI] [燃料費最適化AI] [水力予測AI]                 │
│  [再エネ出力予測AI] [蓄電最適化AI] [配電ロスAI]                │
│  ↑ 時系列ML（LSTM/Transformer）+ 気象データAPI               │
│  ↑ OSIsoft PI / Azure Energy データプラットフォーム            │
├──────────────────────────────────────────────────────────────┤
│  Layer C: OT/設備AI（制御系隣接・最高安全基準）                  │
│  [設備予知保全AI] [停電復旧AI支援] [ドローン×AI点検]            │
│  [系統制御AI支援] [デジタルツイン]                             │
│  ↑ OT専用AIプラットフォーム（GE Predix, Siemens MindSphere等）│
│  ↑ SCADA/DCSとの直接制御接続は禁止。データ読取専用ブリッジ経由   │
└──────────────────────────────────────────────────────────────┘
              ↕  OT/ITブリッジ（DMZ構成・一方向データ転送）
              ↕  発電所SCADA / 変電所DCS / スマートメーター
              ↕  IEC 62443準拠セキュリティ基盤
```

---

## 4-2. OT/ITブリッジ設計（最重要技術課題）

| 連携対象 | 技術課題 | 対応方針 | 期間・コスト |
|---------|--------|---------|:----------:|
| 発電所SCADA（火力・水力） | OT系は外部接続原則禁止。データ抽出は物理的に一方向のみ | データダイオード（一方向データ転送装置）を設置。読取専用データをIT側に転送 | 4-6ヶ月 / 0.8億円 |
| 変電所DCS | 変電所ごとにプロトコルが異なる（IEC 61850等） | 標準プロトコルへの変換レイヤー（ゲートウェイ）を設置 | 6-9ヶ月 / 1.2億円 |
| スマートメーター | 30分値データは既存基盤あり | 既存データ収集基盤をAIデータレイクに接続 | 2-3ヶ月 / 0.3億円 |
| 再エネ（太陽光・風力） | 分散設置で接続数が多い | 遠隔監視システム（SCADA）経由でデータ一括取得 | 3-4ヶ月 / 0.4億円 |

## 4-3. Build / Buy / Partner 意思決定

| ユースケース | 戦略 | 主な選択肢 |
|-----------|:----:|:--------:|
| 電力需給予測AI | **Build（in-house）** | 四国固有の需給パターンはカスタムモデルが最適 |

| デジタルツイン | **Partner（専門外部）** | 日立エネルギー, ABB, Siemens |

## 4-5. AIセキュリティ10領域（Phase 4担当：領域2・5・7・9）

### 領域2：AIサプライチェーンセキュリティ
| 対策 | 四国電力への適用 | 優先度 |
|-----|------------|:-----:|
| OT系AIベンダーの認証 | GE/Siemens等のOT AIベンダーはIEC 62443準拠を確認。発電所への導入前に安全審査 | ★★★★★ |
| 外部LLMサービス認証 | Claude/Azure OpenAI：発電所制御データを絶対に外部LLMに送信しないことを技術的に担保 | ★★★★★ |
| 気象データAPIの信頼性 | 需給予測AIの入力となる気象データAPIのSLA・冗長性を確認 | ★★★★☆ |

### 領域5：プライバシー保護技術（PETs）
| 技術 | 電力業での適用 |
|-----|------------|
| k-匿名化 | スマートメーターの個人需要パターンを集計AIが参照する際の匿名化 |
| 差分プライバシー | 顧客の電力使用量データのAI学習利用に適用 |

### 領域7：データリネージ
| 対象 | 要件 | ツール |
|-----|-----|:----:|
| 需給予測AI学習データ | 気象データ→需要実績→特徴量変換→学習の全ステップ | MLflow |
| 設備AI学習データ | センサーデータ→前処理→異常ラベル→学習の追跡 | Apache Atlas |

### 領域9：Zero Trust for AI
| 原則 | 適用 | 実装 |
|-----|-----|-----|
| 最小権限 | 発電所OTデータへのアクセスはLayer CのAIのみ。コールセンターAIはOTデータにアクセス不可 | Azure RBAC |
| OT/IT分離 | データダイオードにより発電所からの一方向データ転送のみ許可。AIからOTへの書込み禁止 | 物理的データダイオード |
